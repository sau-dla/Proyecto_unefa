Informacion de elaboracion

El Corazón del Proyecto:

Turismo y Tradición Todo gran sistema necesita contenido que valga la pena leer, y aquí es donde entraron nuestras compañeras de Licenciatura en Turismo. Su labor fue fundamental: se encargaron de la curaduría histórica y cultural de las Plazas Bolívar. No se limitaron a copiar y pegar; investigaron la relevancia de estos espacios, su arquitectura y su importancia social. Ellas nos entregaron la materia prima, asegurándose de que cada dato sobre las plazas fuera preciso y nos transportara a esos lugares. Sin su ojo experto para el patrimonio, la hemeroteca solo sería una cáscara vacía.

El Motor Digital:

Análisis y Diseño de Sistemas Por otro lado, el equipo de Sistemas nos encargamos de transformar esa montaña de información en una experiencia digital funcional. Nuestra misión fue estructurar los datos y construir la plataforma desde cero. Para asegurar la máxima calidad, pasamos la información por un filtro de verificación cruzada, utilizando herramientas de IA y Google para corroborar datos históricos y asegurar que lo que el usuario lee sea 100% verídico. Nosotros pusimos la lógica, el código y la arquitectura para que la navegación sea fluida y el sistema soporte todo el contenido de manera eficiente.

La Fusión:

Paso a Paso de Nuestra Hemeroteca ¿Cómo logramos que el Turismo y la Programación hablaran el mismo idioma? Trabajando en un entorno de desarrollo robusto bajo el sistema operativo Ubuntu (Linux), logramos que ambas carreras se complementaran en un proceso que nos tomó aproximadamente dos meses y medio de trabajo intenso:

Fase 1: Maquetación y Estructura: Utilizando Visual Studio Code como nuestra mesa de trabajo, empezamos creando una base sólida con HTML y CSS. Diseñamos una interfaz limpia para que la información turística fuera la protagonista.

Fase 2: Interactividad y Lógica: Implementamos JavaScript para darle vida y "visión" a la página. Queríamos que la navegación no fuera aburrida, sino una experiencia dinámica donde el usuario pudiera explorar las plazas de forma fluida.

Fase 3:Gestión de Datos y Veracidad: Para organizar toda la información recolectada por las chicas y verificada por nosotros, configuramos una base de datos en PostgreSQL. Esto nos permitió manejar los textos y registros de manera profesional y segura.

Fase 4:El Toque Final: Una vez que el sistema estaba estable, integramos las imágenes de referencia y el contenido histórico final, logrando una hemeroteca virtual completa, veraz y funcional.

Dato curioso: Elegir Linux para desarrollar no solo nos dio estabilidad, sino que nos permitió entender mejor cómo funcionan los servidores donde hoy viven este tipo de proyectos culturales.
